# 🚀 Кыргыз Тамыр Календары - Бесплатное Руководство по Развертыванию (Free Deployment Guide)

Этот проект состоит из двух частей:
- **Бэкенд (Backend)**: Python + FastAPI
- **Фронтенд (Frontend)**: React + TypeScript + Vite

Обе части можно абсолютно бесплатно развернуть (deploy) и сделать доступными для всего интернета.

---

## Шаг 1: Подготовка к деплою (GitHub)
Вам нужно загрузить исходный код на [GitHub](https://github.com/). 
Развертывание будет происходить автоматически из вашего репозитория.

1. Зарегистрируйтесь на **GitHub** (если еще нет).
2. Создайте новый пустой репозиторий (New Repository).
3. Откройте терминал в корневой папке вашего проекта (`/home/student/projects/hackaton/`), инициализируйте Git и отправьте код:
   ```bash
   git init
   git add .
   git commit -m "Initial commit for Kyrgyz Calendar"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЮЗЕРКЕЙМ/ИМЯ_РЕПОЗИТОРИЯ.git
   git push -u origin main
   ```

## Шаг 2: Развертывание Бэкенда (Render.com)
Мы будем использовать **[Render.com](https://render.com/)**, так как он предлагает отличный бесплатный тариф для Python-сервисов (идеально для FastAPI).

1. Перейдите на **Render.com** и зарегистрируйтесь через GitHub.
2. Нажмите **"New +" -> "Web Service"**.
3. Подключите ваш GitHub аккаунт и выберите репозиторий с календарем.
4. В настройках заполните:
   - **Name**: `kyrgyz-calendar-api` (или любое другое)
   - **Root Directory**: `Back`  👈 *(очень важно, так как бэкенд лежит в папке Back)*
   - **Environment**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. В разделе **Instance Type** убедитесь, что выбран **Free** (Бесплатно).
6. Нажмите **Create Web Service**.
7. Подождите пару минут, пока Render соберет зависимости. После этого вы получите ссылку на ваш API, например:
   `https://kyrgyz-calendar-api.onrender.com`

## Шаг 2 (Альтернатива): Развертывание Бэкенда на Heroku
Поскольку у вас есть GitHub Student Developer Pack, вы можете развернуть бэкенд на ресурсах **Heroku** совершенно бесплатно с помощью их программы для студентов! Проект уже настроен для Heroku (в корне созданы файлы `Procfile` и `requirements.txt`).

**Как развернуть на Heroku**:
1. Перейдите на **[Heroku Dashboard](https://dashboard.heroku.com/)** и создайте новое приложение: **"New" -> "Create new app"** (например, `kyrgyz-calendar-api`).
2. Во вкладке **Deploy**:
   - В разделе Deployment method выберите **GitHub**.
   - Найдите свой репозиторий с календарем и нажмите **Connect**.
   - Нажмите **Deploy Branch** (ветка `main`).
3. Heroku автоматически найдет новый `Procfile`, перейдет в папку `Back`, установит пакеты через `requirements.txt` и запустит сервер!
4. После деплоя вы получите ссылку: `https://kyrgyz-calendar-api.herokuapp.com`

## Шаг 3: Развертывание Фронтенда (Vercel)
Для React + Vite **[Vercel](https://vercel.com/)** — самый удобный бесплатный хостинг.

1. Зайдите на **[Vercel.com](https://vercel.com/)** и войдите через GitHub.
2. Нажмите **"Add New" -> "Project"**.
3. Выберите ваш GitHub репозиторий проекта и нажмите **Import**.
4. В окне конфигурации настройте следующее:
   - **Framework Preset**: Выберите **Vite**.
   - **Root Directory**: Нажмите "Edit" и выберите папку `Front`. 👈 *(важно!)*
5. **Environment Variables**:
   По умолчанию фронтенд обращается к `localhost:8000`. Нам нужно сказать ему обращаться к вашему новому бэкенду на Render!
   - Откройте вкладку "Environment Variables".
   - Name: `VITE_API_BASE_URL`
   - Value: `https://ваша-ссылка.herokuapp.com/api` *(вставьте сюда ссылку от Render или Heroku, и допишите в конце `/api`)*
6. Нажмите **Deploy**.
7. Подождите минуту! Vercel сгенерирует для вас ссылку на готовый фронтенд.

## 🎉 Готово!
Теперь:
1. Раздайте ссылку от Vercel вашим пользователям или жюри хакатона. Приложение выглядит потрясающе на мобильных устройствах и десктопах!
2. Все запросы интерфейса на астрономические вычисления будут динамически идти к вашему бесплатному серверу на Render.

> **Важно**: Бесплатные серверы на Render "засыпают", если на них нет активности 15 минут. Поэтому при первом открытии сайта утром загрузка календаря может занять 15-30 секунд, пока бэкенд просыпается. Это норма для бесплатного тарифа!
